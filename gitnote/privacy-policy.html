重要提示：Git记 重视你的隐私和个人信息保护。本政策说明 Git记 如何收集、使用、存储、共享、转让、公开披露和保护你的个人信息，以及你如何管理自己的信息。

Git记 是本地优先的 Markdown 笔记应用。通常情况下，我们不运营笔记云服务，不主动收集、上传或读取你的笔记正文。你使用 Git 同步、相机、相册、生物识别、应用内购买、第三方 Git 服务等功能时，相关数据可能由你的设备系统、应用商店、支付渠道或你选择的第三方服务处理。

请在使用 Git记 前仔细阅读本政策。如你不同意本政策，请停止使用 Git记。

一、适用范围

1. 本政策适用于我们提供的 Git记 应用及相关服务，包括可能在 iOS、Android、HarmonyOS、Web、桌面端或其他平台提供的版本。

2. 本政策不适用于你通过 Git记 访问或使用的第三方服务，包括但不限于 GitHub、GitLab、Gitee、自建 Git 服务器、应用商店、云服务、相机、相册、邮件客户端、浏览器、系统分享面板等。第三方服务如何处理你的信息，适用其各自的隐私政策和服务协议。

3. 如果 Git记 后续提供网站、服务器同步、账号系统、客服系统或其他新功能，我们会根据实际情况更新本政策或通过适当方式向你说明。

五、我们如何共享、转让、公开披露信息

1. 共享

我们不会主动向无关第三方出售你的个人信息或笔记内容。以下场景可能涉及信息共享或第三方处理：

  • 你主动将笔记仓库同步至 GitHub、GitLab、Gitee、自建 Git 服务或其他远端仓库；
  • 你通过 App Store、Google Play、华为应用市场、荣耀应用市场、小米应用商店、OPPO 软件商店、vivo 应用商店、支付宝、微信支付或其他平台、应用商店、支付渠道完成购买、订阅、退款或恢复购买；
  • 你通过系统分享、邮件、浏览器、文件应用或其他应用导出、发送或打开内容；
  • 根据法律法规、司法机关、行政机关、监管要求或保护合法权益所必需的场景；
  • 经你另行明确授权或主动选择的其他场景。

2. 转让

如因合并、分立、收购、资产转让、破产清算或类似交易需要转让个人信息，我们会要求新的持有方继续受本政策约束；若处理目的、方式发生重大变化，将依法重新取得你的同意。

3. 公开披露

除你主动公开、法律法规要求、司法或行政机关要求，或为保护你、其他用户、我们及公众的合法权益所必需外，我们不会公开披露你的个人信息。

六、我们如何存储和保护信息

1. Git记 的笔记、仓库、数据库、缓存、配置和凭据主要存储在你的设备本地。你卸载应用、删除仓库、清除系统数据或更换设备可能导致本地数据丢失。

2. 若你启用远程 Git 同步，你的数据会按照你的仓库配置存储到对应的第三方 Git 服务或自建服务器。其存储位置、保存期限、安全措施和访问控制由对应服务决定。

3. 我们会在合理技术范围内采取安全措施保护应用本地数据和凭据处理流程，例如使用系统安全能力、权限控制和本地存储隔离。但没有任何系统或传输方式可以保证绝对安全。

4. 请你妥善保护设备锁屏密码、平台账号、应用商店账号、支付账号、Git 服务账号、访问令牌、SSH 密钥、App 密码和备份文件。因你主动公开、共享设备、越狱、Root、安装不可信软件、误操作或第三方平台安全事件导致的信息泄露风险，由你自行承担。

5. 若发生可能影响你合法权益的个人信息安全事件，我们将根据法律法规要求采取补救措施，并通过应用内通知、邮件、公告或其他合理方式告知。

七、信息保存期限

1. 设备本地信息通常会保存至你主动删除、卸载应用、清除数据、删除仓库或系统清理为止。

2. 你同步到远端 Git 仓库的信息保存期限由你和对应 Git 服务控制。即使你在 Git记 中删除笔记，远端仓库历史提交、分支、标签、备份、缓存或镜像中仍可能保留相关记录，具体取决于 Git 规则和第三方服务规则。

3. 你主动通过邮件或其他方式提交的反馈信息，我们仅在实现反馈处理、争议解决、安全合规或法律要求所必需的期限内保存。

八、你如何管理自己的信息

你可以通过以下方式管理你的信息：

  • 在 Git记 中编辑、删除、移动或恢复笔记；
  • 删除本地仓库、图片附件、最近删除记录、缓存或应用数据；
  • 在 Git 服务平台管理仓库、分支、提交历史、访问令牌、SSH 公钥和成员权限；
  • 在设备系统设置或对应平台权限管理页面中关闭相机、相册、生物识别、通知、网络等权限；
  • 在对应平台账号、应用商店、支付渠道或系统订阅管理页面取消订阅或恢复购买；
  • 通过卸载 Git记 删除应用及其本地容器中的数据，但系统备份、第三方仓库或远端服务中的数据可能不会同步删除；
  • 通过本政策载明的联系方式向我们咨询、投诉或请求协助。

由于 Git 具有版本历史特性，已提交并推送到远端仓库的内容可能需要你在对应 Git 服务中通过删除分支、重写历史、清理缓存或联系服务提供方等方式处理。

九、撤回授权与注销

1. 你可以在设备系统设置或对应平台权限管理页面中撤回相关权限授权。撤回授权后，Git记 将无法继续使用对应权限提供相关功能，但不影响此前基于你的授权已进行的处理。

2. Git记 当前通常不要求注册我们账号，因此不涉及应用内账号注销。你可以通过删除应用、本地仓库、系统数据及第三方 Git 服务中的仓库或凭据，停止继续使用相关功能。

3. 如果你通过邮件向我们提交过反馈或请求，可通过本政策载明的联系方式申请查询、更正或删除你主动提供的信息。为保障安全，我们可能需要核验你的身份或请求来源。

十、未成年人个人信息保护

1. Git记 主要面向具备独立理解和管理 Git 仓库、Markdown 文件及数字内容能力的用户。未成年人使用 Git记 应取得监护人同意，并在监护人指导下使用。

2. 监护人应帮助未成年人正确管理平台账号、应用商店账号、支付方式、Git 服务账号、仓库权限、公开仓库内容和个人信息。

3. 若监护人认为未成年人在未获同意的情况下向我们提供了个人信息，或需要查询、更正、删除相关信息，可通过本政策载明的联系方式联系我们。

十一、跨境处理

Git记 本身主要在设备本地处理数据。若你选择使用境外 Git 服务、应用商店、支付渠道、云服务、自建服务器或其他第三方服务，你的笔记内容、仓库信息、账号信息或交易信息可能根据该服务的规则被传输、存储或处理至境外。你应在使用前了解并同意对应第三方服务的隐私政策和跨境处理规则。

十二、本政策如何更新

1. 我们可能根据法律法规、监管要求、产品功能、运营模式或第三方服务变化更新本政策。

2. 对于重大变更，我们会通过应用内页面、弹窗、公告、邮件或其他合理方式提醒你。重大变更可能包括处理目的、处理方式、个人信息类型、共享对象、用户权利行使方式等发生实质变化。

3. 若你不同意更新后的政策，应停止使用 Git记；若你继续使用 Git记，即表示你已了解并接受更新后的政策。

十四、关键词说明

1. “个人信息”是指以电子或者其他方式记录的与已识别或者可识别的自然人有关的各种信息，不包括匿名化处理后的信息。

2. “敏感个人信息”是指一旦泄露或者非法使用，容易导致自然人人格尊严受到侵害或者人身、财产安全受到危害的个人信息，例如生物识别、精确定位、金融账户、行踪轨迹、未成年人个人信息等。

3. “本地处理”是指相关数据主要在你的设备上完成保存、读取、计算、渲染或管理，不代表数据在任何情况下都不会因你主动同步、导出、分享或使用第三方服务而离开设备。
